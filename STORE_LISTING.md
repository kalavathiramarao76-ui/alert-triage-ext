## Detailed Description

TriageFlow AI is the smartest AI-powered alert triage Chrome extension for SREs, DevOps engineers, and on-call teams. Instantly classify alert priority (P0-P4), detect noise alerts, identify root causes, and get escalation recommendations — right from PagerDuty or OpsGenie.

### Key Features

- **Priority Classification** — AI classifies alerts into P0-P4 severity levels instantly
- **Noise Detection** — Identifies non-actionable alerts to reduce alert fatigue
- **Root Cause Analysis** — AI pinpoints the likely root cause of each alert
- **Escalation Paths** — Recommended team and channel for escalation
- **Incident Categorization** — Automatic category assignment (infra, app, network, etc.)
- **Slack Summary Export** — One-click copy of formatted triage summary for Slack
- **PagerDuty & OpsGenie Integration** — Content script works directly on alerting platform pages
- **Side Panel Workspace** — Full triage workspace with detailed analysis
- **Dark & Light Themes** — Built for on-call engineers at any hour
- **Keyboard Shortcuts** — Ctrl+Enter to triage for fast workflows

### How It Works

1. Click the TriageFlow AI icon or use it on PagerDuty/OpsGenie
2. Paste the alert text or details
3. Click "Triage Alert" (or Ctrl+Enter) for instant classification
4. Review priority, category, root cause, and escalation path
5. Copy the Slack summary for your incident channel

### Use Cases

- On-call SREs triaging alerts during incident response
- DevOps teams reducing alert fatigue with noise detection
- Incident commanders quickly classifying alert severity
- Platform teams standardizing alert triage across the organization
- Teams using PagerDuty or OpsGenie for alerting

### Technical Details

- Manifest V3 Chrome extension (latest standard)
- Integrates with PagerDuty and OpsGenie pages
- AI-powered classification with structured output
- Keyboard shortcut support for fast triage
- Lightweight with minimal permissions

### Privacy

TriageFlow AI sends alert text to our AI API for classification only. Alert data is not stored permanently. No alerting platform credentials are accessed.
